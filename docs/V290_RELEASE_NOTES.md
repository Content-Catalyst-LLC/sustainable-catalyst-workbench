# Workbench v2.9.0 — Technical Documentation and Product Dossier Studio

## New WordPress modules

- `[sc_workbench_documentation_dossier project="default"]`
- `[sc_workbench_product_dossier project="default"]`
- `[sc_workbench_requirements_traceability project="default"]`
- `[sc_workbench_revision_change_control project="default"]`
- `[sc_workbench_verification_evidence project="default"]`
- `[sc_workbench_release_readiness project="default"]`

## Core capabilities

- Generate technical documents from structured live project records.
- Build product dossiers with revision, ownership, source, assumption, limitation, and risk records.
- Evaluate requirements-to-test coverage and identify orphan or failed critical requirements.
- Audit revision histories and engineering change requests.
- Register verification evidence with source, revision, approval, and hash metadata.
- Evaluate category-level manufacturing and release readiness without claiming certification.
- Create versioned documentation snapshots with file manifests and hash-chain fields.
- Export browser-local records as JSON, Markdown, and standalone HTML.
- Discover approved local documentation utilities through the paired Go Runner.

## Compatibility

v2.9.0 preserves the primary `[sc_workbench]` shortcode repair introduced in v2.8.1 and adds Documentation & Dossiers to the unified studio selector.
