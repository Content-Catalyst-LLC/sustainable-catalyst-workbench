# Workbench v4.0.1 Security Boundary

The reliability console may inspect shortcode registration, module rendering state, schema identifiers, project propagation, handoff hashes, asset versions, and integration availability.

It does not execute arbitrary commands, clear caches automatically, rewrite projects, migrate schemas, publish content, operate devices, delete records, or expose private WordPress records. Destructive repair requires a verified backup and explicit human authorization outside the public interface.
