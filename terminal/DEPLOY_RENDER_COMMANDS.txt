# Sustainable Catalyst Workbench v0.7.3 — Render Free deployment

# 1. Push v0.7.3 to GitHub
cd ~/Downloads
chmod +x update_and_push_sustainable_catalyst_workbench_v0_7_3_to_github.sh
./update_and_push_sustainable_catalyst_workbench_v0_7_3_to_github.sh

# 2. In Render, create a Web Service from:
# Content-Catalyst-LLC/sustainable-catalyst-workbench
# Root Directory: backend
# Build Command: pip install --upgrade pip && pip install -r requirements.txt
# Start Command: ./start.sh
# Health Check Path: /health
# Plan: Free

# 3. Free-first Gemini AI path:
# SC_WORKBENCH_AI_PROVIDER=gemini
# GEMINI_API_KEY=<paste-your-gemini-key-in-render>
# SC_WORKBENCH_GEMINI_MODEL=gemini-3.5-flash

# 4. DeepSeek alternate:
# SC_WORKBENCH_AI_PROVIDER=deepseek
# DEEPSEEK_API_KEY=your_deepseek_key
# SC_WORKBENCH_DEEPSEEK_MODEL=deepseek-v4-flash

# 5. Calculator-only fallback:
# SC_WORKBENCH_AI_PROVIDER=disabled

# 6. Test once Render gives you a URL:
curl https://YOUR-RENDER-URL.onrender.com/health
curl https://YOUR-RENDER-URL.onrender.com/ai/status

# 7. In WordPress:
# SC Workbench → Settings
# Backend URL: https://YOUR-RENDER-URL.onrender.com
# AI provider: Use backend .env setting
# Enable AI panels: checked
# Enable scope gate: checked
